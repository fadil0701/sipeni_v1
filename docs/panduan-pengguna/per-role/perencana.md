# Panduan: Perencana

> **Akses di aplikasi:** Menu sidebar **Panduan Pengguna** — bab ini otomatis ditampilkan untuk role Anda.

## Profil role

| | |
|---|---|
| **Level** | Pusat |
| **Modul default** | Planning, Transaksi |
| **Legacy setara** | `perencanaan`, `admin_perencanaan` |

## Kegunaan

Perencanaan kebutuhan — review **RKU**, kelola **Program/Kegiatan/Sub Kegiatan**, disposisi tahap perencanaan.

## Menu utama

| Menu | Kegunaan |
|------|----------|
| **Perencanaan → Program/Kegiatan/Sub Kegiatan** | Master perencanaan |
| **Perencanaan → Daftar RKU** | Review & proses seluruh RKU lintas unit; boleh tambah RKU |
| **Perencanaan → Rekap Tahunan** | Rekap kebutuhan |
| **Approval** | Disposisi terkait perencanaan (jika diizinkan) |

## Alur RKU

1. Terima RKU status **diajukan** / **diproses** dari unit
2. Review kesesuaian kegiatan & prioritas
3. **Start review** / berikan catatan / forward ke TU
4. Monitor RKU **disetujui** → koordinasi pengadaan

## Tips

- Pastikan master Sub Kegiatan sudah update sebelum musim RKU
- Gunakan rekap tahunan untuk analisis gap kebutuhan antar unit
