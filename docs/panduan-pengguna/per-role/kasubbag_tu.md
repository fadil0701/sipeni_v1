# Panduan: Kasubbag TU

> **Akses di aplikasi:** Menu sidebar **Panduan Pengguna** — bab ini otomatis ditampilkan untuk role Anda.

## Profil role

| | |
|---|---|
| **Level** | Pusat |
| **Modul default** | Transaksi, Laporan |

## Kegunaan

Verifikasi **administrasi** permintaan barang dan RKU sebelum naik ke Kepala Pusat; monitoring laporan.

## Menu utama

| Menu | Kegunaan |
|------|----------|
| **Approval** | Verifikasi / kembalikan permintaan |
| **Perencanaan → RKU** | Review tahap TU |
| **Monitoring** | Laporan |
| **Transaksi → Peminjaman** | Diketahui kasubbag (tahap peminjaman) |

## Alur permintaan barang

1. Approval → permintaan status **diajukan** / menunggu TU
2. Periksa kelengkapan: unit, pemohon, jenis, item, qty
3. **Verifikasi** → lanjut kepala pusat
4. **Kembalikan** → unit perbaiki (status kembali ke pemohon)

## Alur RKU

1. Daftar RKU → filter review kasubbag
2. Periksa kesesuaian program/kegiatan & anggaran
3. Approve review atau minta revisi

## Tips

- Gunakan filter tanggal & unit di Approval untuk antrian harian
- Catat alasan saat **kembalikan** agar unit tidak bingung
